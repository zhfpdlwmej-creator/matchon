# matchon 운영 전략

## 1. 컨셉 재확인 — "보조 서비스"
카카오톡을 대체하지 않는다. 모든 운영 메시지는 **"단톡방으로 돌아가게"** 설계한다.
- 초대: 앱이 만든 6자리 코드/링크를 **단톡방에 붙여넣기**
- 알림: 카카오 채널/알림톡으로 전달 → 클릭하면 참석 버튼
- 즉, matchon 은 단톡방의 "참석 집계 봇" 역할.

## 2. 출시(Go-to-Market)
### 초기 타깃
- 운영 고통이 큰 **20~100명 규모, 주 1회 이상 정기전** 동호회 우선
- 시드: 지인 동호회 2~3팀 직접 온보딩(총무를 챔피언으로)

### 온보딩 플로우
1. 총무가 팀 생성 → 초대코드 단톡방 공유
2. 첫 정기전 1건 등록 → 알림 발송 체험
3. 1주 사용 후 피드백 → 개선

### 바이럴 루프
- 초대코드 = 자연 확산. "옆 동호회도 쓰자" 유도
- 출석왕/통계 캡처 공유(카톡 공유 카드, V1)

## 3. 카카오 알림 운영
| 단계 | 방식 | 비고 |
|------|------|------|
| MVP | `mode=log` + 단톡방 수동 공유 링크 | 채널 승인 전 |
| V1 | 카카오 **알림톡**(비즈채널+템플릿 승인) | 정보성 메시지, 야간 발송 정책 준수 |
| 대안 | 친구톡/문자(LMS) 폴백 | 알림톡 실패 시 |
- 템플릿 사전 신청(승인 3~7일) → 4주차에 착수
- 발송 시간대: 30분전 알림은 야간(21시~08시) 정책 확인, 과발송 방지(동일 type 1회)

## 4. 인프라 & 비용
- **MVP**: 단일 인스턴스(PaaS: Railway/Render/클라우드런) + MariaDB(매니지드) → 월 1~3만원
- 백업: DB 일 1회 자동 스냅샷, Flyway로 스키마 버전관리
- 도메인/HTTPS 필수(카카오 Redirect, 쿠키 Secure)
- 스케일: 팀·트래픽 증가 시 인스턴스 수평 확장(무상태 JWT라 용이), 읽기 부하는 인덱스/캐시

## 5. 보안 & 개인정보
- 수집 최소화: 카카오 식별자 + 닉네임만(이메일/전화 미수집)
- JWT 비밀키/카카오 시크릿은 환경변수, 코드/깃 비포함(`application-local.yml` gitignore)
- 권한 서버검증(컨트롤러에서 `requireManager`), WEB-INF로 뷰 직접접근 차단
- 탈퇴/데이터 삭제 요청 대응 절차 마련(CASCADE 삭제)

## 6. 지표 모니터링
| 지표 | 목표 | 용도 |
|------|------|------|
| 팀당 주간 참석 응답률 | ≥70% | 핵심 가치 작동 여부 |
| 일정 등록→첫 응답 시간 | 단축 | 알림 효과 |
| 4주 팀 리텐션 | ≥60% | PMF 신호 |
| 알림 발송 성공률 | ≥98% | 채널 건강도 |
- 로그 기반 대시보드 + 주간 리뷰

## 7. 고객 지원/피드백
- 인앱 "건의하기" → 운영 채널, 총무 대상 1:1 채팅(카톡 채널)
- 변경 로그/공지(가벼운 릴리스 노트)

## 8. 수익화(검토, 비강제)
- 기본 무료(소규모 팀)
- 프리미엄(팀 단위 월 구독): 알림톡 무제한, 회비 장부, 통계 고급, 용병 모집 — V2 이후
- 광고/제휴: 풋살장 예약·용품(맥락 광고) — 신중히

## 9. 로드맵 요약
- **MVP**: 참석 관리 + 기본 알림(log/공유)
- **V1**: 알림톡 실연동, 통계/회비/랭킹, 카톡 공유 카드
- **V2**: 정기 일정 반복, 용병 모집, 자동 팀 밸런싱, 결제·장부, PWA/푸시
