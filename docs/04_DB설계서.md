# matchon DB 설계서 (MariaDB · utf8mb4)

엔진 InnoDB, 문자셋 utf8mb4. 실제 DDL 은 `src/main/resources/db/migration/V1__init.sql` (Flyway).

---

## USERS — 사용자
| 컬럼 | 타입 | Null | 키 | 설명 |
|------|------|------|----|------|
| id | BIGINT AI | N | PK | 사용자 PK |
| kakao_id | VARCHAR(64) | N | UK | 카카오 회원번호(고유) |
| nickname | VARCHAR(20) | N | | 서비스 내 닉네임 |
| position | VARCHAR(8) | Y | | 선호 포지션 GK/DF/MF/FW |
| profile_image | VARCHAR(255) | Y | | 프로필 이미지 URL |
| setup_done | TINYINT(1) | N | | 최초 닉네임 설정 완료 여부(default 0) |
| created_at | DATETIME | N | | 가입일시 |
| updated_at | DATETIME | N | | 수정일시(ON UPDATE) |

- INDEX: `uq_users_kakao_id(kakao_id)` UNIQUE

## 설계 노트
- **모든 PK BIGINT AUTO_INCREMENT** — 단순/성능. 외부 노출 식별자(초대코드)는 별도 컬럼.
- **스키마는 Flyway가 단일 소스** — JPA `ddl-auto=validate` 로 엔티티-스키마 불일치 조기 검출.
- **삭제 정책** — 팀/일정 삭제 시 하위(멤버·참석·댓글·알림) CASCADE 로 정리.
- **참석 미응답 처리** — ATTENDANCE 행이 없는 멤버는 애플리케이션에서 PENDING(미정)으로 간주(전 멤버 행 선생성 안 함 → 쓰기 비용↓).
- **타임존** — 앱/JVM `Asia/Seoul`, DATETIME 저장.
