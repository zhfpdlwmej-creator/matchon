# matchon MVP 우선 개발 순서

"가장 빠르게 가치를 검증"하는 순서. 각 단계는 **독립적으로 동작/시연 가능**하게 끊었다.

## 원칙
1. 인증 → 팀 → 일정 → 참석 순으로, **참석 체크가 동작하는 최단 경로**를 먼저 완성
2. 화면은 "뼈대(JSP) + 데이터(AJAX)"로 분리해 백엔드 먼저, UI 덧붙이기
3. 알림/통계/회비는 코어 검증 후 (없어도 참석 관리는 성립)

## 단계별 순서

### STEP 1. 기반 (없으면 아무것도 안 됨)
1. 프로젝트 셋업(build.gradle, application.yml)
2. DB 스키마(Flyway V1) + 엔티티 7종
3. JWT + SecurityConfig + 카카오 로그인(+mock) + `/welcome`
> ✅ 검증: 카카오로 로그인 → 닉네임 설정 → 세션 유지

### STEP 2. 팀 (참석을 담을 그릇)
4. 팀 생성 → 생성자 LEADER
5. 초대코드 발급/가입
6. 팀원 목록 + 권한
> ✅ 검증: A가 팀 생성·코드 공유 → B가 가입 → 팀원 2명

### STEP 3. 일정 (참석 대상)
7. 일정 등록(운영진 권한 가드) — 경기명/날짜/시간/장소/참가비/메모
8. 일정 목록 + 월별 달력
9. 일정 상세
> ✅ 검증: 운영진이 일정 등록 → 달력/목록/상세 노출

### STEP 4. 참석 (★핵심 가치★)
10. 참석/불참/미정 토글 (unique 1인1응답)
11. 참석 현황 집계(참석n/불참n/미정n) + 명단
12. 홈 "가장 가까운 경기" + 원터치 참석
> ✅ 검증: 회원들이 버튼 탭 → 실시간 집계 → **여기까지가 MVP의 본질**

### STEP 5. 보강 (경험 완성)
13. 포지션별 인원 표시
14. 일정 댓글
15. 알림(log) — 일정등록 즉시 + 리마인드 스케줄러

### STEP 6. 운영 도구 (출시 직전)
16. 출석률 통계(QueryDSL)
17. 회비 납부 체크/미납 조회
18. 인원부족 알림 + 관리자 화면

## "여기서 멈춰도 출시 가능" 라인
- **STEP 4 완료 = 최소 출시 가능**(참석 관리 보조라는 약속을 지킴).
- STEP 5~6 은 리텐션/운영 편의를 높이는 증분.

## 절대 MVP에 넣지 않는 것 (의도적 보류)
- 실시간 채팅, 결제(PG), 자동 팀 밸런싱, 정기 일정 반복, 푸시/PWA → V1/V2
- 이유: 카카오톡 보조 컨셉 유지 + 출시 속도. 범위 확장은 검증 후.
